# G1.1 — 建立可运行骨架

> GitHub Issue: [#2](https://github.com/cosZone/koharu-suite/issues/2)

## 前置

- Roadmap #1 已确认。
- Node.js 22、pnpm workspace、PostgreSQL 18、Drizzle ORM、Hono、React + Vite、Vitest 已确定。
- 本 Goal 从空仓库开始，不依赖其他实现 Goal。

## 背景

后续 Telegram、认证、文章和媒体能力需要共享一套可安装、可测试、可构建和可部署的承载面。本 Goal
先打通 server、Admin、CLI、数据库迁移、测试、Docker 与 CI 的最小闭环，避免之后分别搭建互不一致的
横向脚手架。

## 范围

- pnpm workspace 与统一工程命令。
- 可测试的 Hono health endpoint。
- 可构建并能读取 health 的 React/Vite Admin 壳。
- 单一 `kodama` CLI 的 help/version/server/migration 入口。
- PostgreSQL 18 + Drizzle schema、生成 migration 和带 advisory lock 的执行器。
- Vitest unit 与 Testcontainers PostgreSQL 18 integration smoke。
- Node.js 22 Dockerfile、Compose 与 GitHub Actions CI。
- 开发 quick start 与 `.env.example`。

## 交付物

- `apps/server`
- `apps/admin`
- 根 workspace、TypeScript、Biome、Vitest、Docker 与 CI 配置
- 首个已提交的 Drizzle migration
- 本 Goal 文档

## 验收

- 根目录命令可以完成 lint、typecheck、test、integration test 与 build。
- server 暴露 `/healthz` 和 `/api/v1/health`。
- Admin 开发模式能显示 server health，生产构建成功。
- `kodama --help`、`kodama --version`、`kodama serve` 和 `kodama migrate` 入口存在。
- migration 在真实 PostgreSQL 中幂等执行，并由 advisory lock 串行化。
- Docker image 能构建，Compose 配置有效。
- CI 使用准确 pnpm 版本和 frozen lockfile。
- Compose、Testcontainers 与 CI 使用同一个 `postgres:18-alpine` 镜像基线。

## 验证命令

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm test:integration
pnpm build
pnpm exec kodama --help
docker compose config
docker build .
```

## 文档影响

- README 增加开发 quick start。
- `.env.example` 只列出非敏感示例。
- 后续业务 Goal 应沿用本 Goal 的 workspace、migration、测试和容器入口。

## 明确不做

- Telegram Bot、频道与消息 schema。
- Better Auth、owner、session 与 service token。
- 搜索、RSS、媒体下载或 storage adapter。
- 正式 Admin UI、共享组件设计系统或 Storybook。
- Astro adapter、npm 发布与生产部署。
